# Storyboard SOP

This file explains how to use the storyboard folder during image generation.

## Purpose
- `storyboard/ACT*/board*.md` remains the master shot list.
- `storyboard/ACT*/board*/` stores generation working files for that board.
- Generated images, selected finals, and prompt notes should live beside the board they belong to.

## Recommended Workflow
1. Lock character anchors first.
2. Read the matching `board*.md` file.
3. Copy the shot prompt into the board working folder.
4. Generate a small batch first.
5. Review continuity, character stability, and lighting consistency.
6. Keep only the usable versions and note what failed.

## Folder Convention
- `storyboard/ACT1/board1/prompts.md`
  Stores copy-ready prompts, batch notes, status, and chosen versions.
- `storyboard/ACT1/board1/images/`
  Stores generated images for that board only.

## Naming Convention
- Use shot-first names so files stay sortable.
- Example:
  - `1.1_v1.png`
  - `1.1_v2.png`
  - `1.3_selected.png`
  - `1.9_alt_red-eye.png`

## What To Track
- whether the character face is stable
- whether the bookstore / street matches the visual bible
- whether the shot function is readable
- whether the image is reusable for animatic / motion
